Abstract —For deep learning inference on edge devices, hard-ware configurations achieving the same throughput can differ by 2× in power consumption, yet operators often struggle to find the efficient ones without exhaustive profiling. Existing approaches often rely on inefficient static presets or require expensive offline profiling that must be repeated for each new model or device. To address this problem, we present C ORAL , an online optimization method that discovers near-optimal configurations without offline profiling. C ORAL leverages distance covariance to statistically capture the non-linear dependencies between hardware settings, 

e.g. , DVFS and concurrency levels, and performance metrics. Unlike prior work, we explicitly formulate the challenge as a throughput-power co-optimization problem to satisfy power budgets and throughput targets simultaneously. We evaluate CORAL on two NVIDIA Jetson devices across three object detection models ranging from lightweight to heavyweight. In single-target scenarios, C ORAL achieves 96% – 100% of the optimal performance found by exhaustive search. In strict dual-constraint scenarios where baselines fail or exceed power budgets, CORAL consistently finds proper configurations online with minimal exploration. 

Index Terms —Edge Computing; Deep Learning Inference; DVFS Optimization; Resource Allocation; Distance Covariance 

I. I NTRODUCTION 

Real-world deployments of edge applications, e.g. , from autonomous drones to smart surveillance, demand consistent inference throughput under strict power constraints [1]. For example, a drone running object detection must sustain a sufficient frame rate for responsive navigation while staying within a power budget that preserves battery life (and sufficient flying time as well). Both constraints – throughput and power requirements – must be satisfied simultaneously. However, achieving this balance is a non-trivial research problem [2]. The core difficulty lies in the sheer number of possible configurations on modern edge hardware. Edge devices like NVIDIA Jetson have dozens of tunable parameters, including CPU and GPU frequencies, memory clock rates, and active core counts [3], [4]. However, interactions among these param-eters are complex and exhibit non-linear behavior. Changing one parameter affects the impact of others, making it dif-ficult to predict performance from individual settings alone. Manufacturers simplify this complexity by offering a handful of preset power modes, but these presets represent only a

Moreover, both approaches typically optimize for a single metric, either throughput or power, leaving dual-constraint scenarios largely unaddressed. The irregular search space, where many config-urations satisfy one constraint but violate the other, defeats conventional optimization strategies [7]. We present C ORAL (Covariance-Guided Resource Adaptive Learning), an online optimization method that discovers near-optimal configurations on edge devices by exploiting the following key insight. The statistical relationship between hardware setting changes and their performance impact can be captured through distance covariance, which does not require exhaustive profiling. Distance covariance [8] measures dependency between vari-ables regardless of their relationship’s functional form, mak-ing it ideal for the non-linear configuration space of edge devices. C ORAL computes correlation weights between each hardware parameter (CPU frequency, GPU frequency, memory clock, core count, concurrency level) and both target metrics (throughput and power). These weights guide a lightweight search that prioritizes high-impact parameters, converging to valid configurations within 10 iterations, orders of magnitude faster than profiling-based alternatives. Unlike prior work [6], [9] that optimizes for throughput alone or power alone, C ORAL 

explicitly formulates the problem as throughput-power co-optimization. By maintaining separate correlation weights for throughput and power, C ORAL identifies configurations that satisfy both constraints simultaneously. We evaluate C ORAL on NVIDIA Jetson Xavier NX and Orin Nano across three object detection models: YOLOv5-N, FRCNN-MobileNetV3, and RetinaNet-ResNet50. We compare against four state-of-the-art baselines: ORACLE (exhaustive offline profiling of all configurations), ALERT (offline profil-ing with Kalman filtering) [6], ALERT-Online (online adap-tation of ALERT), and manufacturer presets ( max-power 

and default modes). Our evaluation results show that when optimizing for throughput alone, CORAL achieves 96% – 100% of ORACLE performance. The more challenging case is optimizing for both power and throughput simultaneously. Under a 6.5 W power limit and 30 fps throughput target on Xavier NX with YOLOv5-N, C ORAL achieves 34 fps at 5.9 W,satisfying both constraints. In contrast, ALERT exceeds the power budget at 8.5 W, ALERT-Online fails to find valid configurations, and the default preset delivers only 15 fps. These results hold across all model sizes and both devices. This paper makes the following contributions:  

> •

Online Optimization Without Offline Profiling: We introduce the use of distance covariance for hardware configuration search, enabling real-time adaptation with-out offline profiling or expensive model training. To our knowledge, CORAL is the first method to apply this statistical technique to edge inference optimization.  

> •

Throughput-Power Co-Optimization Formulation: We formalize edge inference as a throughput-power co-optimization problem with explicit power limits and throughput targets. This addresses real deployment con-straints but has been largely ignored by prior work.  

> •

Comprehensive Evaluation: We evaluate C ORAL on two real-world NVIDIA Jetson devices across three object detection models spanning 20 × size difference, demon-strating consistent improvements over baselines in both single-metric and dual-constraint scenarios. II. B ACKGROUND AND RELATED WORK 

A. Background 1) Hardware Configuration on Edge Devices: Modern edge AI platforms provide fine-grained control over hardware settings through Dynamic Voltage and Frequency Scaling (DVFS) [3]. NVIDIA Jetson devices exemplify this capability. Jetson Xavier NX (X AVIER -NX) [10], built on the Volta architecture, features six Carmel CPU cores, a 384-core GPU with 48 Tensor cores, and 8GB LPDDR4X memory. The newer Jetson Orin Nano (O RIN -N ANO ) [11] uses the Ampere architecture with six Cortex-A78AE cores, a 1024-core GPU with 32 Tensor cores, and 8GB LPDDR5 memory. Table 1 summarizes their specifications. On these platforms, operators can adjust multiple parameters that directly affect both performance and power consumption:

CPU frequency and core count: Higher frequencies and more active cores increase computational throughput but also raise power draw.  

> •

GPU frequency: Important configuration for DL infer-ence, as most neural network operations execute on the GPU.  

> •

Memory frequency: This configuration affects data transfer rates between memory and processing units.  

> •

Concurrency level: The number of inference instances running simultaneously [12], [13]. Higher concurrency can improve hardware utilization by overlapping CPU preprocessing with GPU computation, but also increases resource contention and power consumption. B. Related Work 1) Hardware-Aware Optimization: A substantial body of work addresses hardware efficiency for DL inference. DVFS and power management strategies have been extensively stud-ied to navigate the fundamental trade-off between power and performance. 

Profiling-based methods. ALERT [6] demonstrated the effec-tiveness of coordinating application and system-level adapta-tion, achieving significant energy savings through probabilistic modeling with Kalman filters. However, ALERT requires extensive offline profiling that must be repeated for each model-device combination. 

Learning-based methods. Zhang et al. [3] proposed DVFO, using deep reinforcement learning (DRL) to co-optimize DVFS parameters for edge-cloud collaborative inference. While effective, DRL-based methods require extensive training iterations and may struggle to converge quickly in new envi-ronments. Yan et al. [4] introduced PolyThrottle, applying con-strained Bayesian optimization for energy-efficient inference, but such methods may still need offline exploration phases. 

Online (or offline profiling-free) methods. SHEEO [14] uses reinforcement learning to continuously optimize energy effi-ciency without prior profiling, monitoring runtime variations, and adapting power management strategies accordingly. CORAL differs from these approaches by using distance covariance for lightweight online search, requiring no offline profiling, no training phase, and explicitly targeting dual-constraint scenarios where both power and throughput must be satisfied. 

2) Concurrent Execution Frameworks: Recent frameworks address execution orchestration for edge inference. Miriam [15] proposes a contention-aware task coordination framework for multi-DNN inference on edge GPUs, achieving significant throughput improvements while maintaining low latency for critical tasks. HaX-CoNN [16] enables heterogeneity-aware execution of concurrent DNNs by intelligently scheduling workloads across diverse accelerators within a System-on-Chip. Hao et al. [13] explored maximizing inference through-put by leveraging AI multi-tenancy and dynamic model placement on heterogeneous edge devices. C ORAL comple-ments these frameworks by functioning as a device-level optimizer. While frameworks like HaX-CoNN manage dis-tributed scheduling, C ORAL dynamically tunes the underlying hardware parameters (concurrency level and DVFS settings) in real-time to meet specific throughput and power targets. III. T HE CORAL METHOD 

A. Problem Formulation 

We aim to find a hardware configuration that satisfies both throughput and power constraints. We define the configuration space as: 

s = ( scpu , c cpu , s gpu , s mem , c ) (5) where scpu is CPU frequency, ccpu is the number of active CPU cores, sgpu is GPU frequency, smem is memory fre-quency, and c is concurrency level. Each configuration s yields a throughput τ (s) and power consumption p(s).

Throughput-Power Co-Optimization. Given a throughput target τtarget and power limit pbudget , we seek a configuration 

s∗ that satisfies both constraints: 

τ (s∗) ≥ τtarget and p(s∗) ≤ pbudget (6) 3A configuration that violates either constraint is marked as infeasible and excluded from future exploration. Among all feasible configurations, we prefer those with higher efficiency 

η(s) = τ (s)/p (s).

Search Strategy. CORAL adapts its search direction based on the current configuration’s performance. If the throughput target is already satisfied, C ORAL attempts to reduce power consumption to improve efficiency. Otherwise, C ORAL prior-itizes increasing throughput to first reach the feasible region. This adaptive strategy enables C ORAL to navigate toward the feasible region while optimizing for efficiency. 

Challenges. This problem has three challenges. First, the configuration space can be large (over 2K combinations per device in our experiments). Second, the functions τ (s) and 

p(s) are non-linear and exhibit complex interactions among parameters as discussed in §I. Third, the feasible region satisfying both constraints is often narrow, making exhaustive search impractical. These challenges motivate C ORAL ’s dis-tance covariance-based approach, which efficiently identifies high-impact parameters without exhaustive profiling. 

B. Overview 

CORAL operates in three steps at each iteration. 

Step 1: Reward Evaluation (§III-C). CORAL executes inference with the current configuration and measures throughput and power. It then evaluates the results using Algorithm 1 to find 1) configurations satisfying both con-straints receive a positive reward (efficiency) and 2) those violating constraints receive a penalty and are added to a prohibited list. The best and second-best configurations are updated based on the reward. 

Step 2: Correlation Analysis (§III-D). CORAL computes distance correlation between each hardware parameter and both performance metrics (throughput and power) us-ing recent observations. This identifies which parameters have the strongest influence on performance. 

Step 3: Configuration Search (§III-E). CORAL uses the correlation weights to determine the next configuration to explore. Parameters with higher correlation receive larger adjustments, while less influential parameters change minimally (Algorithm 2). These steps repeat for a fixed iteration budget (10 in our implementation). 

C. Reward and Feasibility Check 

Algorithm 1 evaluates each configuration by checking fea-sibility and computing a reward score. 

Feasibility Check. A configuration is feasible if it satisfies both constraints: throughput meets the target ( τ ≥ τtarget ) and power stays within the budget ( p ≤ pbudget ). Configurations violating either constraint are marked as infeasible and added to a prohibited list ( P S ). Before exploring a new configura-tion, C ORAL checks P S and skips any previously identified infeasible configurations to avoid redundant evaluations.

Interpretation. A high αi indicates that parameter i strongly affects throughput; a high βi indicates strong influence on power. In the decision phase, C ORAL takes the dominant correlation γi = max( αi, β i) and uses it to scale adjustments, such that parameters with higher correlation receive larger ad-justments, while less influential parameters change minimally. 

E. Configuration Search 

Algorithm 2 generates the next configuration based on correlation weights and the current best configurations. 

Step Size. For each parameter dimension i, C ORAL com-putes a combined weight γi = max( αi, β i) and calculates a weighted step size: 

∆i = 12 |xi − yi| · γi (10) where xi and yi are the values of parameter i in the best and second-best configurations, respectively. This step size is proportional to both the distance between good configurations and the correlation strength. Parameters with higher correlation receive larger adjustments, while less influential parameters change minimally.

A. Experimental Setup 

We describe the DL (object detection) models, baselines, and evaluation procedures used in our evaluations. 

DL models. We evaluate C ORAL using three object detec-tion models with varying computational demands (Table 3): YOLOv5-N (YOLO) with 1.9M parameters [17], FRCNN-MobileNetV3 (FRCNN) with 19.4M parameters [18], [19], and RetinaNet-ResNet50 (R ETINA NET ) with 38M parame-ters [20], [21]. This selection spans a 20 × range in model size, allowing us to evaluate C ORAL ’s effectiveness across lightweight to heavyweight inference workloads. The PyTorch framework is used to train these models on 80 object classes from the COCO dataset [22]. All models use an input size of 640 × 640. Among the three, R ETINA NET achieves the highest accuracy (41.5 of mAP), followed by FRCNN (32.8 of mAP) and YOLO (27.6 of mAP), consistent with their respective model size and capacities. 

Measurement Setup. We evaluate C ORAL on two NVIDIA Jetson devices: X AVIER -NX and O RIN -N ANO (specifications in Table 1 in §II-A). We limit CPU usage to 90% using 

cgroups to prevent resource contention. To collect system-level metrics, we use tegrastats 1 (for power and GPU utilization/metrics) and sysstat 2 package (CPU and mem-ory utilization/metrics). Measurements begin 2 seconds after inference starts and update every second, synchronized with throughput measurements.

Evaluation Procedure. We implement all optimization meth-ods within a common evaluation loop that applies configu-rations, executes inference, and collects performance metrics. The resulting performance metrics, throughput (fps) and power consumption (m W), are measured and fed back to the op-timizer for evaluation. This feedback loop continues for a fixed number of iterations. We set the iteration budget to 10, which balances search effectiveness with practical deployment constraints. This budget is sufficient for C ORAL to converge while remaining feasible for online adaptation scenarios.
