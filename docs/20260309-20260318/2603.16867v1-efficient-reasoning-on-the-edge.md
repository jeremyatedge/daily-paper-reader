## 摘要
具有思维链（CoT）推理能力的大语言模型（LLMs）在处理复杂问题解决任务时表现卓越，但其冗长的推理轨迹和庞大的上下文需求使其难以在边缘端部署。这些挑战包括高昂的 Token 生成成本、巨大的 KV 缓存占用，以及将推理能力蒸馏至移动端小模型时的低效。现有方法通常依赖于将大模型的推理轨迹蒸馏到小模型中，但这些轨迹往往冗长且风格冗余，不适合端侧推理。本研究提出了一种轻量级方法，通过结合 LoRA 适配器与有监督微调，使小参数 LLM 具备推理能力。我们进一步在这些适配器上通过强化学习引入了预算强制（budget forcing）机制，在精度损失极小的情况下显著缩短了响应长度。针对受内存限制的解码问题，我们利用并行测试时扩展，在延迟略微增加的前提下提升了准确率。最后，我们提出了一种动态适配器切换机制（仅在需要时激活推理）以及 Prompt 编码阶段的 KV 缓存共享策略，从而降低了端侧推理的首字延迟。在 Qwen2.5-7B 上的实验表明，我们的方法在严格的资源限制下实现了高效、准确的推理，使 LLM 推理在移动场景中具有实用性。项目主页提供了该方案在移动设备上运行的演示视频。

## Abstract
Large language models (LLMs) with chain-of-thought reasoning achieve state-of-the-art performance across complex problem-solving tasks, but their verbose reasoning traces and large context requirements make them impractical for edge deployment. These challenges include high token generation costs, large KV-cache footprints, and inefficiencies when distilling reasoning capabilities into smaller models for mobile devices. Existing approaches often rely on distilling reasoning traces from larger models into smaller models, which are verbose and stylistically redundant, undesirable for on-device inference. In this work, we propose a lightweight approach to enable reasoning in small LLMs using LoRA adapters combined with supervised fine-tuning. We further introduce budget forcing via reinforcement learning on these adapters, significantly reducing response length with minimal accuracy loss. To address memory-bound decoding, we exploit parallel test-time scaling, improving accuracy at minor latency increase. Finally, we present a dynamic adapter-switching mechanism that activates reasoning only when needed and a KV-cache sharing strategy during prompt encoding, reducing time-to-first-token for on-device inference. Experiments on Qwen2.5-7B demonstrate that our method achieves efficient, accurate reasoning under strict resource constraints, making LLM reasoning practical for mobile scenarios. Videos demonstrating our solution running on mobile devices are available on our project page.

### 1. 核心问题与整体含义（研究动机和背景）
*   **核心问题**：大型语言模型（LLM）虽然通过思维链（CoT）推理在复杂任务上表现优异，但其推理过程极其冗长（Token 数量多）、KV 缓存占用巨大且计算成本高，导致其难以部署在资源受限的边缘设备（如手机）上。
*   **研究背景**：现有的推理模型蒸馏方法往往产生风格冗余的输出。在移动端，内存带宽是主要瓶颈，且并非所有用户查询都需要复杂的推理。因此，如何在严格的内存、延迟和功耗预算下，赋予端侧小模型高效的推理能力是本文的研究重点。

### 2. 论文提出的方法论
该论文提出了一个端到端的推理优化框架，包含以下关键技术：
*   **模块化推理适配器（LoRA + SFT）**：使用低秩适配器（LoRA）对基础指令模型进行有监督微调（SFT），使其具备推理能力。这种方式允许模型在普通聊天模式和推理模式之间动态切换。
*   **预算强制强化学习（Budget Forcing RL）**：采用 GRPO 算法，引入一种**乘法形式的软屏障奖励函数**：$R = R_{accuracy} \times R_{budget}(L)$。该函数会惩罚超过预设长度（如 1k, 3k tokens）的回复，强制模型在保证正确性的前提下压缩推理轨迹。
*   **动态路由切换器（Switcher）**：在模型顶层添加一个轻量级 MLP 分类器，在 Prefill（预填充）阶段判断查询复杂度。简单问题直接由基础模型回答，复杂问题才激活推理 LoRA。
*   **掩码 LoRA 预训练（Masked LoRA）**：在训练推理适配器时，对 Prefill 阶段的 LoRA 权重进行掩码处理，使得推理模式能直接复用基础模型的 KV 缓存，避免了切换模式时的重复编码延迟。
*   **并行测试时缩放（Parallel TTS）与轻量级验证器**：利用解码阶段的内存受限特性，并行生成多个推理流，并使用一个仅增加极少参数的线性验证器头（Verifier Head）进行加权投票，提升准确率。
*   **量化感知推理（QAMR）**：采用 W4A16KV8 量化方案（4位权重、16位激活、8位 KV 缓存），并结合函数保持变换（FPTQuant）来减少量化带来的精度损失。

### 6. 主要结论与发现
*   **推理压缩**：通过预算强制 RL，在几乎不损失准确率的情况下，推理轨迹长度平均缩减了 **2.4 倍**，最高可达 8 倍。

*   **路由效率**：Switcher 能有效识别简单问题，通过跳过推理适配器大幅节省了非必要计算。

### 7. 优点（亮点）
*   **端到端实用性**：不仅关注算法，还解决了 KV 缓存共享、首字延迟（TTFT）等实际部署中的痛点。
*   **创新的奖励设计**：乘法奖励函数有效解决了 RL 过程中的“奖励作弊”（Reward Hacking）问题，防止模型通过格式手段规避长度惩罚。
*   **硬件感知**：充分利用了移动端 NPU/GPU 在解码时的算力冗余进行并行推理。

### 8. 不足与局限
*   **任务遗忘风险**：SFT 过程虽然提升了复杂推理能力，但在简单的代码生成任务（如 HumanEval）上出现了性能退化，存在“过专业化”导致通用能力下降的问题。
*   **Switcher 局限性**：目前的切换器基于监督学习，未来若能通过 RL 学习更优的路由策略（如根据推理密度路由）会更具潜力。
*   **极低比特挑战**：目前主要研究 4-bit 权重，对于更极端的 2-bit 或 3-bit 量化，模型推理能力的保持仍是挑战。
