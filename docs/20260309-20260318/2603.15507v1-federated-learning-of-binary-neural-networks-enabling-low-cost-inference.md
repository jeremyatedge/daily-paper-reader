## 摘要
联邦学习 (FL) 通过在设备间分布式进行训练来保护隐私。然而，在推理过程中，在低功耗边缘设备上使用深度神经网络 (DNN) 会产生巨大的计算开销。边缘部署要求模型同时优化内存占用和计算效率，而传统 DNN 往往因超出资源限制而无法满足这一需求。传统的训练后二值化虽然减小了模型大小，但由于量化误差，会导致严重的精度损失。为了应对这些挑战，我们提出了 FedBNN，这是一个旋转感知的二值神经网络框架，它在本地训练过程中直接学习二值表示。通过将每个权重编码为单个比特 {+1, -1} 而非 32 位浮点数，FedBNN 缩小了模型占用空间，与使用实值模型的联邦方法相比，显著降低了推理时的浮点运算量 (FLOPs) 和内存需求。在多个基准数据集上的评估表明，FedBNN 在显著降低资源消耗的同时，其性能与现有的使用实值模型的联邦方法相当。

## Abstract
Federated Learning (FL) preserves privacy by distributing training across devices. However, using DNNs is computationally intensive at the low-powered edge during inference. Edge deployment demands models that simultaneously optimize memory footprint and computational efficiency, a dilemma where conventional DNNs fail by exceeding resource limits. Traditional post-training binarization reduces model size but suffers from severe accuracy loss due to quantization errors. To address these challenges, we propose FedBNN, a rotation-aware binary neural network framework that learns binary representations directly during local training. By encoding each weight as a single bit $\{+1, -1\}$ instead of a $32$-bit float, FedBNN shrinks the model footprint, significantly reducing runtime (during inference) FLOPs and memory requirements in comparison to federated methods using real models. Evaluations across multiple benchmark datasets demonstrate that FedBNN significantly reduces resource consumption while performing similarly to existing federated methods using real-valued models.

    *   **现有 FL 压缩不足**：现有的联邦学习压缩方法（如 FedBAT、FedMUD）大多侧重于减少“通信开销”，而最终部署在设备上的模型往往仍是实值（Real-valued）模型，推理成本依然很高。

### 2. 核心方法论：FedBNN 框架
FedBNN 的核心思想是在联邦训练过程中直接学习二值化表示（权重为 +1 或 -1），并通过旋转感知技术弥补量化损失。

*   **旋转感知训练（Rotation-Aware Training）**：
    *   引入旋转矩阵 $R_1$ 和 $R_2$，通过最小化实值权重向量与二值化向量之间的夹角（Angular Bias）来降低量化误差。
    *   在每个本地训练周期开始时，通过交替优化（SVD 分解）更新旋转矩阵。
*   **全局与本地知识融合**：
    *   使用可训练参数 $\lambda$ 融合本地权重与服务器下发的全局权重。
    *   引入可学习标量 $\alpha$ 和 $\beta$，动态调节旋转方向、全局模型方向对最终二值化权重的贡献。
*   **训练感知近似（Training-aware Approximation）**：
    *   由于 `sign` 函数不可导，采用一种随训练轮数演变的平滑代理函数（Surrogate Function）来替代，确保训练初期梯度流动顺畅，后期逐渐收敛至标准的二值化操作。
*   **服务器端聚合策略**：
    *   服务器维护两套表示：一套用于通信的实值聚合权重，另一套用于评估和模型选择的“旋转对齐”权重。

*   **端到端二值化**：不同于只压缩通信的方法，FedBNN 产出的直接就是可部署的轻量化二值模型。
*   **旋转优化创新**：将旋转感知技术引入联邦学习，有效解决了分布式环境下的量化偏置问题。
*   **参数自适应**：$\lambda, \alpha, \beta$ 均为自动学习，无需人工繁琐调参。

### 8. 不足与局限
*   **精度损失**：尽管表现优异，但在复杂任务（如 Tiny-ImageNet）上，与全精度模型相比仍有明显的精度缺口。
