## 摘要
大语言模型（LLMs）正越来越多地作为 AI 智能体（AI agents）部署，它们在简短的“推理-行动”循环中运行，将模型计算与外部调用交织在一起。与传统的聊天应用不同，这些智能体工作负载要求推理服务系统在来自不同 AI 智能体的多个请求并发到达时，平衡低延迟、稳定的 Token 发射和吞吐量。受隐私、合规性和成本约束的驱动，最近的部署呈现出在消费级 GPU 上本地运行小语言模型（SLMs）的趋势。当异构请求在单个 GPU 上重叠时，长预填充（prefills）和短解码（decodes）会争夺资源，产生队头阻塞（head-of-line blocking），从而破坏交互性能的稳定性。通过分析智能体工作负载，我们观察到它们的执行自然地分为：处理长系统提示词的冷预填充（cold prefills）、将工具输出附加到缓存上下文的恢复预填充（resume prefills），以及对延迟敏感的短解码。与传统的聊天机器人服务相比，这种混合负载加剧了资源竞争。我们提出了 AgentServe，这是一个单 GPU 服务系统，通过将预填充与解码隔离、对恢复预填充应用动态预算，并通过具有自适应控制的预设 CUDA Green Context 插槽分配 GPU 资源，确保在此类条件下多智能体执行的稳定性。评估结果表明，AgentServe 在保持竞争性吞吐量的同时，显著提高了延迟稳定性，在不同设置下，相比最先进的基准方案，实现了高达 2.8 倍的 TTFT 提升和 2.7 倍的 TPOT 提升。

## Abstract
Large language models (LLMs) are increasingly deployed as AI agents that operate in short reasoning-action loops, interleaving model computation with external calls. Unlike traditional chat applications, these agentic workloads require inference serving systems to balance low latency, stable token emission, and throughput under multiple request arrivals from different AI agents. Recent deployments highlight a shift toward running small language models (SLMs) locally on consumer-grade GPUs, driven by privacy, compliance, and cost constraints. When heterogeneous requests overlap on a single GPU, long prefills and short decodes contend for resources, creating head-of-line blocking that destabilizes interactive performance. By analyzing agent workloads, we observe that their execution naturally separates into cold prefills, which process long system prompts, resume prefills, which append tool outputs to cached contexts, and short decodes, which are latency-critical. This mix intensifies contention compared to conventional chatbot serving. We present AgentServe, a single-GPU serving system that ensures stable multi-agent execution under such conditions by isolating prefills from decodes, applying dynamic budgeting to resume prefills, and allocating GPU resources through pre-established CUDA Green Context slots with adaptive control. Evaluation results show that AgentServe significantly improves latency stability while sustaining competitive throughput, achieving up to 2.8x TTFT improvement and 2.7x TPOT improvement over state-of-the-art baselines across different settings.

#### 1. 核心问题与研究动机
随着 AI 智能体（AI Agents）的兴起，推理负载从传统的“长对话”转向了“短推理-行动循环”。智能体工作负载具有独特的模式：包含极长的系统提示词（冷预填充）、频繁追加工具输出（恢复预填充）以及极短且对延迟敏感的结构化输出（短解码）。
在**消费级 GPU**（如 RTX 系列）上本地部署小语言模型（SLM）时，长预填充任务会长时间占用计算资源，导致严重的**队头阻塞（HoL Blocking）**，使得并发执行的解码任务出现明显的延迟抖动（TPOT 飙升），破坏了交互的流畅性。现有的系统（如 vLLM、SGLang）多针对数据中心 GPU 或长文本生成优化，难以在单块消费级 GPU 上完美平衡预填充与解码的冲突。

#### 2. 方法论
AgentServe 采用了**算法与系统的协同设计**，核心思想是实现预填充与解码的深度隔离与动态资源平衡：
*   **三阶段负载感知：** 将任务细分为**冷预填充**（处理长系统提示词）、**恢复预填充**（追加工具返回的少量 Token）和**短解码**。
*   **TPOT 驱动的资源调度算法：** 引入反馈控制机制。系统实时监测解码阶段的“每 Token 生成时间（TPOT）”。如果 TPOT 超过阈值，则增加解码任务的 GPU 流处理器（SM）预留比例，并缩减允许与解码并发执行的预填充 Token 预算；反之则释放资源给预填充任务。
*   **CUDA Green Context 资源隔离：** 在系统底层利用 NVIDIA 的 Green Context 技术，预设 10 个不同 SM 比例（10%-100%）的插槽。通过轻量级的线程重绑定，实现预填充和解码在单 GPU 上的空间隔离，避免了多进程通信或频繁创建上下文的开销。
*   **共享内存管理：** 预填充和解码线程共享 GPU 内存池，确保 KV Cache 在不同阶段间无缝传递，无需跨设备或跨进程拷贝。

#### 7. 优点
*   **协同设计思路清晰：** 紧扣智能体负载的特殊性（冷/热预填充之分），将算法层面的调度与底层的硬件隔离技术完美结合。

#### 8. 不足与局限
*   **硬件依赖：** CUDA Green Context 是较新版本 CUDA 和特定架构 GPU 的特性，在旧型号硬件上可能无法直接应用。
*   **模型规模限制：** 实验主要集中在 3B-8B 的小模型，对于需要多卡并行的超大模型（如 70B+），该单卡优化方案的扩展性有待验证。
